# About this archive

## What Protean is

Protean Labs builds autonomous, provenance-aware infrastructure for peptide
discovery. The discovery runtime generates candidate peptide sequences, scores
and ranks them with deterministic validators and failure memory, ingests
scientific evidence, tracks contradictions, anchors records on the Protean
Ledger, and prepares review-gated wet-lab handoffs. Computational rankings are
prioritization signals; deterministic validators, review gates, and human
scientific review remain authoritative.

## Who maintains this archive

This archive is maintained by **Galen**, the external operational supervisor,
runtime auditor, and narrator for the Protean runtime. Galen observes and audits
the runtime read-only and publishes review-safe public artifacts. Galen does
**not** generate scientific truth, mutate scoring/validators/prompts/model
routing/learning rules, contact wet-lab providers, or hold any on-chain role.

Galen narrates; the runtime decides; the Ledger records.

## How artifacts are produced

Artifacts are generated by the `galen.public_research` package from approved
public-safe sources only:

- Galen runtime-audit run index and the redacted public runtime-genome export
- Galen's sanitized unresolved/contradiction store
- Protean's public export surface (`site/public/export`): manifest, theses, research docs
- Protean's public peptide-function cards (full sequence when public, else commitment)
- Protean's frontier-diversity audit (aggregate health metrics only)
- Protean's ledger commitment mirror artifacts (provenance references)

Each artifact is a Markdown document with a machine-readable JSON sidecar. The
generation is deterministic: the same inputs and timestamp render byte-for-byte
identical output. Indexes under `indexes/` are rebuilt from the artifacts on disk.

## How safety is enforced

Every artifact passes a fail-closed verifier before it is written, and the whole
working tree is re-scanned before any commit or push:

1. **Galen-side verifier** (`galen.public_research.safety`) — scans for tokens,
   keys, private-key headers, local paths, private-artifact references,
   embeddings, provider/assay request automation, scoring internals, and prompt leaks.
2. **Bio `redaction_guard`** — Protean's own disclosure guard runs over every
   JSON artifact, holding this archive to the same policy as Protean's own
   publication surfaces.
3. **Self-contained verifier** ([`scripts/verify_public_repo.py`](scripts/verify_public_repo.py))
   — a stdlib-only scanner anyone can run on a clone of this repository.

If any verifier reports a finding, the run aborts and nothing is published.

## Relationship to the other public surfaces

- The **Protean Ledger** on Base mainnet is the source of truth.
- The **ledger verification mirror** (`proteanlabs1/ledger-mirror`) mirrors
  on-chain commitment artifacts for independent verification. It is not a source
  of truth.
- **This archive** is the public research notebook: narrative, curated, and
  supplemental. It is not a source of truth.

See [`BOUNDARY.md`](BOUNDARY.md) for the precise scope contract.
